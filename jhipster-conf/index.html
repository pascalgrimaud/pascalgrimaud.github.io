## What do we need to test ? - the generator him self - the installation of tools: node, yarn, java, git, etc. - the generated application - backend tests - frontend tests - e2e tests
## Hard to test everything - front: angular / react - back: maven / gradle - security: jwt, oauth2, session - css: sass - cache: no l2, - database: mysql, mariadb postgresql, mongodb, cassandra, couchbase - option: kafka, elasticsearch, websocket - microservices: eureka, consul, uaa, oauth2 - lot of options: more than 20000 different configs - if you think about entities, it will be more / more
## Daily builds hipster-labs - Docker - Maven - Gradle - Microservices - React
## Tips for contributor As contributor, you can have your own Travis builds The workflow: - work on your fork - push in a branch to your work - a Travis builds will be trigger in your Travis - if everything passed, you can pull request
